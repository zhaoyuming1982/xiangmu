# 项目说明与使用手册

## 一、项目主要内容

本项目是一个 A 股日线行情数据自动更新程序，主要功能是通过 baostock 获取股票基础信息和日线行情数据，结合本地行业分类文件生成下载任务，并将数据写入本地 MySQL 数据库。

程序会自动完成以下工作：

- 初始化 MySQL 数据库 `stock_db`。
- 创建并维护股票日线主表 `stock_daily`。
- 生成或更新股票下载任务文件 `stock_task.csv`。
- 启动 10 个 worker 脚本并行下载股票日线数据。
- 执行兜底全量下载，减少并发下载遗漏。
- 计算日内涨跌幅、振幅、涨停价、跌停价、涨跌停状态、连续涨停天数、量比、今日量比、流通市值等衍生指标。
- 将涨停和跌停记录同步到 `stock_limit` 表。
- 按行业自动拆分数据表，便于后续按行业查询。
- 对未知行业股票尝试从同花顺页面补齐行业信息，并记录到 `industry.txt`。

- `main.py` 中使用的是固定本地路径，如 `C:\xiangmu\stock_task.csv`，移动项目目录后需要同步修改路径配置。

## 三、运行环境

### 1. 基础环境

- Windows 系统
- Python 3.11 或兼容版本
- MySQL 服务已启动
- 可以访问 baostock 数据接口
- 如需自动补齐未知行业，需要能访问同花顺页面

### 2. Python 依赖

程序主要依赖：

```bash
pip install baostock pandas pymysql openpyxl
```

其中：

- `baostock`：获取股票基础信息和历史行情。
- `pandas`：处理 CSV、Excel 和任务数据。
- `pymysql`：连接并写入 MySQL。
- `openpyxl`：读取 Excel 文件，供转换脚本使用。

### 3. MySQL 配置

当前数据库连接配置写在 `main.py` 和各 `worker*.py` 文件中：

```python
MYSQL_USER = "root"
MYSQL_PASSWORD = "Root123456"
MYSQL_HOST = "localhost"
MYSQL_PORT = 3306
MYSQL_DB = "stock_db"
```

使用前请确认：

- MySQL 用户名和密码正确。
- MySQL 服务正在运行。
- 当前用户有创建数据库、创建表、插入、更新、删除和创建索引的权限。

## 四、数据库表说明

程序会自动创建和维护以下表：

| 表名 | 作用 |
| --- | --- |
| `stock_daily` | 股票日线主表，保存行情数据和计算后的衍生指标。 |
| `stock_limit` | 涨跌停明细表，只保存涨停或跌停记录。 |
| 行业表 | 按 `industry` 字段动态创建，一行业一张表，结构与 `stock_daily` 基本一致。 |
| `industry_sync_state` | 行业表同步水位表，用于增量同步行业数据。 |

主要业务字段包括：

| 字段 | 含义 |
| --- | --- |
| `date` | 交易日期 |
| `code` | 股票代码，如 `sh.600000` |
| `code_name` | 股票名称 |
| `industry` | 所属行业 |
| `open`、`high`、`low`、`close` | 开盘价、最高价、最低价、收盘价 |
| `volume`、`amount`、`turn` | 成交量、成交额、换手率 |
| `pctChg` | 涨跌幅 |
| `intra_pct` | 日内涨跌幅 |
| `amplitude` | 振幅 |
| `up_limit`、`down_limit` | 涨停价、跌停价 |
| `limit_status` | 涨跌停状态：0 普通，1 涨停，2 跌停 |
| `limit_days` | 连续涨停天数 |
| `volume_ratio` | 5 日均量比 |
| `vol_ratio_today` | 今日成交量与昨日成交量比值 |
| `circ_market` | 流通市值估算 |

更完整的建表 SQL 可查看 `userdate.md`。

## 五、首次使用步骤

### 1. 检查 MySQL

确认 MySQL 已启动，并且 `main.py` 中的连接配置可以正常登录。

### 2. 检查行业文件

确认根目录下存在：

```text
C:\xiangmu\stock_industry.csv
```

该文件用于给股票匹配行业。如果行业缺失，程序会标记为 `未知行业`，并尝试从同花顺补齐。

### 3. 安装依赖

在命令行执行：

```bash
pip install baostock pandas pymysql openpyxl
```

### 4. 运行主程序

在项目根目录或任意目录执行：

```bash
python C:\xiangmu\更新程序\更新程序\main.py
```

程序会依次执行：

1. 初始化数据库和基础表。
2. 初始化 `stock_task.csv`。
3. 根据数据库已有数据更新续传日期。
4. 启动 10 个并行下载脚本。
5. 再次更新续传日期并重复并行下载。
6. 执行兜底下载。
7. 计算所有衍生指标。
8. 修正未知行业。
9. 同步涨跌停表。
10. 同步行业拆分表。

执行完成后，控制台会输出“全部任务完成”。

## 六、日常更新步骤

日常更新时，一般只需要重新运行主程序：

```bash
python C:\xiangmu\更新程序\更新程序\main.py
```

程序会根据 `stock_daily` 中每只股票已有的最大日期，自动更新 `stock_task.csv` 的 `start_date`，实现断点续传，避免重复下载已存在的数据。

## 七、Excel 转行业 CSV

如果需要从 Excel 重新生成 `stock_industry.csv`，可使用辅助脚本：

```bash
python C:\xiangmu\转换文件转换前为xlsx转换为csv.py
```

注意：

- 当前脚本默认读取同目录下的 `333.xlsx`。
- 导出文件名为 `stock_industry.csv`。
- 脚本会强制将股票代码按文本读取，避免 `00` 开头代码丢失前导零。
- 如实际 Excel 文件名不是 `333.xlsx`，需要先修改脚本中的文件名。

## 八、日志与排查

常用排查文件：

| 文件 | 作用 |
| --- | --- |
| `download_progress.log` | 记录 worker 下载过程和进度。 |
| `industry.txt` | 记录未知行业股票的补齐结果。 |
| `stock_task.csv` | 查看每只股票当前下载任务、起止日期和行业。 |

常见问题：

### 1. MySQL 连接失败

检查：

- MySQL 服务是否启动。
- `MYSQL_USER`、`MYSQL_PASSWORD`、`MYSQL_HOST`、`MYSQL_PORT` 是否正确。
- 当前用户是否有建库和建表权限。

### 2. baostock 登录或下载失败

可重新运行主程序。worker 内置了登录重试和超时关闭机制，主程序也会执行兜底下载。

### 3. 部分股票行业显示未知行业

检查 `stock_industry.csv` 是否包含对应股票代码。程序会尝试从同花顺补齐，但网络不可用或页面结构变化时可能失败。

### 4. 运行路径改变后报找不到文件

项目中多个脚本使用固定路径 `C:\xiangmu`。如果移动项目目录，需要同步修改 `main.py`、`worker*.py` 和转换脚本中的路径。

## 九、维护建议

- 修改 MySQL 配置时，需要同时检查 `main.py` 和 `worker1.py` 至 `worker10.py`。
- 不建议手动删除 `stock_daily`，否则会导致断点续传依据丢失。
- 若需要重跑全部历史数据，可清空相关数据库表，并删除或重新生成 `stock_task.csv`。
- `stock_industry.csv` 是行业拆表的基础，更新前建议备份。
- 大批量下载前建议确认磁盘空间、MySQL 可用空间和网络稳定性。
